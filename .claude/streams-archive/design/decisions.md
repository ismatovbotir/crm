# Design Stream — Decisions Log

Append-only лог архитектурных решений по дизайну. Каждая запись = дата + решение + причина.

---

## 2026-04-27 — Light theme (а не dark/auto)
**Решение**: Используем только light theme в первой версии.
**Причина**: B2B CRM, основное использование днём в офисе. Dark theme — Phase 2+.
**Источник**: лид проекта, основано на референсе react-free.tailwind-admin.com.

## 2026-04-27 — Tailwind CSS v3 (а не v4)
**Решение**: Tailwind v3.4 + PostCSS + autoprefixer.
**Причина**: v4 ещё новый, плагины (forms, typography) более стабильны на v3. Совместимость с Laravel ecosystem.

## 2026-04-27 — Inter font + system fallback
**Решение**: Inter через Google Fonts + system-ui fallback.
**Причина**: универсальный sans-serif, хорошо читается в таблицах и формах.

## 2026-04-28 — Двухуровневая навигация: sidebar + sticky header
**Решение**: Левый сайдбар (фиксированный 256px) + верхний header (64px) с поиском.
**Причина**: стандартный паттерн B2B-админок (Filament, Laravel Nova). Информационная плотность.

## 2026-04-28 — Slide-over для форм (а не modal или отдельная страница)
**Решение**: Создание/редактирование сущностей открывается в slide-over панели справа.
**Причина**: пользователь не теряет контекст списка. Modal слишком ограничен по высоте, отдельная страница теряет состояние списка.

## 2026-04-28 — Permission-aware UI через App\Helpers\Acl
**Решение**: Все условные рендеры в Blade — через `@acl('permission')` директиву или `\App\Helpers\Acl::can()`.
**Причина**: Единый способ во всём проекте, работает в preview-mode (без Spatie) и production. См. CLAUDE.md → "Контроль доступа".
